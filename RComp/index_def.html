Краткая справка по работе с бесплатным хостингом от h18.ru

Также рекомендуем посмотреть ЧАВО.

Доступ к серверу
Для доступа к серверу вам надо использовать любой FTP клиент, позволяющий работать с сервером в активном режиме. В параметрах соединения необходимо указывать хост: ftp.rcomp.h18.ru

Для пользователей ваш сервер будет виден по адресу http://rcomp.h18.ru или http://www.rcomp.h18.ru
Ваш сервер может быть недоступен некоторое время (~ сутки) т.к. необходимо время для того, чтобы он был прописан во всех DNS.

Баннеры
Баннеры автоматически вставляются в страницы вашего сайта.

Структура сервера
В вашей домашней директории находятся поддиректории "WWW" & "cgi".
В "WWW" должны находиться ваши html файлы. Именно она представляет собой содержимое вашего веб сервера. Например, файл begin.html, находящийся в директории "WWW", будет доступен через веб как "http://rcomp.h18.ru/begin.html".

Директория "cgi" предназначена для cgi & SSI скриптов вашего сервера. ВНИМАНИЕ! Для директории "cgi" права доступа должны быть выставлены как 711. Она видна через веб как /cgi-bin/ . Например, файл test.cgi, лежащий в директории "cgi", будет виден как "http://rcomp.h18.ru/cgi-bin/test.cgi".
Не забудьте, что для работы скриптов у них должен быть выставлен атрибут "executable". Для этого вам необходимо выполнить команду 'chmod * 700' или установить аттрибуты всех файлов и директорий находящихся в "cgi" равными "rwx------" с помощью вашего FTP клиента.
Обратите внимание на то, что наш сервер не даст запускать скрипты из директории "WWW" и не будет показывать ваши страницы находящиеся в директории "cgi".

Java апплеты должны находиться в директории "WWW"

Все файлы на сервере должны быть в кодировке win-1251. Если посетители вашего веб сайта запросят файлы в какой-либо другой кодировке, то они будут автоматически перекодированы нашим веб сервером (у нас установлен Russian Apache).

Размер дискового пространства и разрешенные форматы файлов
Основная политика администрации h18.ru состоит в следующем:

  • первоначально Вам отдаётся под сайт 100 Мб;
  • эти 100 Мб дискового пространства могут быть использованы под хранение любых данных за исключением файлов следующих форматов: exe rar gz tar bz2 bz tgz lha lzh arj mov avi mp3 mpg mpeg wav au qt mpe midi ra ram rm.
    "Неправильные" файлы будут удаляться без предупреждения;
  • также вводится ограничение на размер файлов разрешенных форматов: файл не может превышать в размере 10Mb;
  • при широкой популярности Вашего сайта и высокой посещаемости оного, Вам может быть выделено дополнительное пространство.

Серверные логи
Вы можете получить доступ к серверным логам следующим образом: в своем домашнем каталоге создайте директорию с именем "log". Она должна находиться на одном уровне с каталогами WWW и cgi. В ней создайте файл "access" для access.log. Обратите внимание на то, что файл не имeeт расширения. Ежесуточно лог-файлы за прошедшие сутки будут архивироваться в файлы с именем вида accessYYYYMMDD. Лог архивы старше десяти дней будут удаляться.

Кэширование документов на сервере
На машине обслуживающей веб хостинг установлен прокси, кэширующий статические документы, отдаваемые сервером.
Это позволяет увеличить скорость отдачи документов, одновременно существенно снижая нагрузку на сервер. Теперь все часто запрашиваемые статические документы отдаются непосредственно из кэша прокси без запроса к веб серверу.

Документы кэшируются прокси сервером в течение одного часа. Т.е. если вы выложили на сервер документ, то пользователи его увидят с небольшой задержкой (от нескольких минут до одного часа).
Кэшируются все статичные HTML документы. Кэширование не производится для динамически создаваемых страниц, таких как результат работы CGI и PHP скриптов.

Если вам необходимо чтобы пользователи увидели обновление немедленно, то вы можете воспользоваться скриптом очищающим кэш прокси http://register.h18.ru/purge.shtml. Правила пользования написаны на странице.
ВНИМАНИЕ! Проблемы с кэшированием наблюдаются только у пользователей Internet Explorer. Пользователи Netscape Navigator могут просто нажать RELOAD, и страница будет удалена из кэша.

SSI
SSI поддерживается для всех файлов с расширением ".shtml".

Проблемы с сервером
В случае возникновения у вас каких-либо проблем с вашим сервером вы можете связаться с нашей службой поддержки по адрес hs@agava.com
В письме обязательно указывайте свой логин (rcomp8) или полное доменное имя (rcomp.h18.ru).

В случае возникновения каких-либо проблем с доступом к серверу вы должны обязательно прислать нам tracert до вашего сервера. Его можно получить командой tracert rcomp.h18.ru. Например, "tracert rcomp.h18.ru" или tracert сразу в файл "tracert rcomp.h18.ru >out.txt"
В письме необходимо указать дату и время (московское), в которое был сделан trace.

Редиректор
После окончания переноса файлов на наш сервер вы должны поставить редирект на новый адрес на старом месте. Для редиректа используйте следующий html:

<html>
<head>
<title>We've Moved!</title>
<META HTTP-EQUIV="Refresh" CONTENT="0; URL=http://rcomp.h18.ru">
</head>
<body></body>
</html>
